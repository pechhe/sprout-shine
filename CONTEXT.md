# Sprout Shine

A voice-first adaptive maths tutor for 7–10-year-olds. Lessons are generated and
delivered by an LLM, but always constrained by enforced structural and safety
rules. Parents receive a weekly learning insight.

## Language

**Lesson Plan**:
The schema-valid description of one lesson — its objective, ordered phases, tasks,
hint ladders, misconception tags, mastery check, and reflection. Generated by the
LLM, then validated and cached before use.
_Avoid_: lesson script, curriculum, syllabus

**Tutor Move**:
A single action the tutor takes in one turn (`say`, `ask`, `give_hint`,
`acknowledge_attempt`, `reveal_worked_step`, `advance_phase`, `reflect_prompt`,
`end_lesson`). It is the unit the LLM emits and the system validates before applying.
_Avoid_: message, response, reply

**Verdict**:
The deterministic correctness result (correct / partial / incorrect) for a child's
attempt, computed by a validator — never by the LLM.
_Avoid_: grade, score, mark

**Hint Ladder**:
The ordered hints for a task, least to most revealing. The deepest hint is a worked
step, never a bare answer.
_Avoid_: hints list, clues

**Misconception**:
A named, controlled-vocabulary error pattern a child's attempt reveals (e.g.
`rows_columns_confused`).
_Avoid_: mistake, error, bug

**Mastery Check**:
The task requiring the child to prove understanding of the objective, not just say a
number.
_Avoid_: test, quiz, final question

**Reflection**:
The child's end-of-lesson response about what helped them learn.
_Avoid_: feedback, review

**Skill Tag**:
A controlled-vocabulary identifier for a maths skill the learner model tracks (e.g.
`multiplication_as_groups`).
_Avoid_: topic, subject

**Workspace**:
The visual manipulative area where a child places objects (counters, groups, arrays)
to solve a task. Its state is machine-inspectable. "Workbook" is the child-facing
label for the same thing.
_Avoid_: canvas, scratchpad

**Answer Type**:
How a task is answered and validated: `manipulative` (Workspace inspector),
`numeric` (parsed number), `choice` (exact match), or `explanation` (captured as
evidence, not deterministically graded).
_Avoid_: input type, response type

**Learner Model**:
A child's persistent, humble set of working hypotheses about their skills,
misconceptions, and learning patterns, built from evidence across sessions. It is
never a set of fixed labels or a diagnosis. It has a level and a confidence for
each hypothesis so weak signals are not overstated.
_Avoid_: profile, learner profile, student record, diagnosis

**Skill State**:
One hypothesis in the Learner Model for a single Skill Tag: an estimated level, a
confidence, the number of evidence points seen, and when it was last seen. Updated
from both diagnostic and lesson evidence.
_Avoid_: score, grade, skill rating

**Pattern Signal**:
A working hypothesis in the Learner Model about *how* a child learns (e.g.
benefits_from_visuals, rushes_when_confident, avoids_explaining) as opposed to a
Skill State, which is about *what* they can do. Like a Skill State it carries a
confidence so it is not treated as a fixed trait.
_Avoid_: trait, personality label, learning style

**Session Event**:
An immutable record of one thing that happened in a session (an attempt graded,
a hint shown, a phase changed). The raw log from which evidence is derived.
_Avoid_: log line, telemetry

**Skill Outcome**:
The normalized unit a learner-model update consumes for a Skill Tag: the verdict,
attempts taken, whether a hint was used, and the phase. Built deterministically
from one Session Event. Distinct from the raw event and from the Skill State it
updates.
_Avoid_: result, attempt record

**Pattern Observation**:
The normalized unit a learner-model update consumes for a Pattern Signal: a tag,
whether the evidence supports or argues against it, a source, and a confidence.
Several build a Pattern Signal, the way several Skill Outcomes build a Skill State.
_Avoid_: pattern instance, behavior log
